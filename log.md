## [2026-04-07] query | How does 5QLN map to LLM Wiki?

**Asked by:** Amihai
**Pages consulted:**
- `SCHEMA.md`
- `evolution.md` (absorbed)
- Karpathy's LLM Wiki gist (read via read_webpage)

**Synthesis:**
5QLN already implements the LLM Wiki pattern — but without the persistent wiki layer.

The mapping:
- **Raw sources** → `Datasets/` (SparkWell, SFF, FCF docs) + `sources/` directories
- **The wiki** → `5qln-journal/` entries + `evolution.md` + session files (partially — these are session outputs, not the persistent cross-linked wiki)
- **The schema** → `AGENTS.md` + `SOUL.md` + phase skills (these ARE the schema, but they're documents about the grammar, not a maintained wiki about the grammar's evolution)

The gap: the journal entries are **session records** but not **entity/concept pages** that get updated across sessions. The wiki doesn't yet maintain cross-references, track contradictions, or file query answers back as new pages.
